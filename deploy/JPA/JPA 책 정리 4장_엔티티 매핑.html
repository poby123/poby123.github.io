# 엔티티 매핑
  @Entity
  JPA를 통해서 DB의 테이블과 매핑할 클래스는 @Entity 를 붙여야한다.

    주의사항
    1. protected 또는 public 기본생성자가 있어야한다.
    2. final 클래스, enum, interface, inner 클래스에는 사용할 수 없다.
    3. 저장할 필드에 final을 사용하면 안된다.

  @Table
  엔티티와 매핑할 테이블을 지정한다. 
  생략하면 엔티티 이름을 테이블 이름으로 사용한다.


# 기본키 전략
  @Id
  기본키는 직접 할당할 수도 있지만, DB가 할당해주는 값을 쓸 수도 있다. 이렇게 하려면 어떻게 해야할까?

  1. IDENTITY (mySql, PostgreSQL, SQL Server, DB2)
     @GenereatedValue(strategy = GenerationType.IDENTITY)  로 지정하면 된다.
    DB에 엔티티가 저장된 이후에 Id가 할당되기 때문에, 추가로 DB를 조회하게 된다.
    DB에 저장되기 전에는 Id가 할당되지 않으므로, 트랜잭션을 지원하는 쓰기 지연이 동작하지 않는다.

  2. SEQUENCE (Oracle, PostgreSQL, DB2, H2)
    em.persist() 를 호출할 때, 먼저 DB 시퀀스를 통해 Id를 조회하고 엔티티에 할당한 다음, 영속 상태로 만든다.
    따라서 Identity 전략과 달리 트랜잭션을 지원하는 쓰기 지연이 동작하여, 나중에 커밋해서 flush가 일어날 때 DB에 저장된다.

    이 전략을 사용하려면, 먼저 DB의 시퀀스를 생성해야한다.
     CREATE SEQUENCE {SEQUENCE_NAME} START WITH {START_NUMBER} BY {INCREMENT_NUMBER} 
    
    그 다음에는 엔티티와 이 시퀀스를 매핑해서 Id를 할당한다.
    @Entity
    @SequenceGenerator(
        name="BOARD_SEQ_GENERATOR",
        sequenceName="SEQUENCE_NAME",
        initialValue = START_NUMBER,
        allocationSize = INCREMENT_NUMBER,
    )
    public class Board {
        @Id 
        @GeneratedValue(
            strategy = GenerationType.SEQUENCE, 
            generator = "BOARD_SEQ_GENERATOR"
        )
        private Long id;
    }


  3. TABLE
    키 생성 전용 테이블을 하나 만들어서 DB 시퀀스를 흉내내는 전략이다.

    이 전략을 사용하려면, 먼저 키 생성 전용 테이블을 만들어야한다.
     CREATE table MY_SEQUENCES( 
         sequence_name varchar(255) not null, 
        next_val bigint,
        primary key ( sequence_name )
    )

    
    그 다음에는 엔티티와 이 시퀀스를 매핑해서 Id를 할당한다.
    @Entity
    @TableGenerator(
        name = "BOARD_SEQ_GENERATOR",
        table = "MY_SEQUENCES",
        pkColumnValue = "BOARD_SEQ",
        allocationSize = 1
    )
    public class Board {
        @Id 
        @GeneratedValue(
            strategy = GenerationType.TABLE, 
            generator = "BOARD_SEQ_GENERATOR"
        )
        private Long id;
    }

  4. AUTO
  데이터베이스 방언에 따라 위 세가지 방법 중 하나를 자동으로 선택한다.


# 필드와 컬럼 매핑
  @Column 
  객체 필드를 테이블 컬럼에 매핑한다.
  - name : 매핑할 테이블의 컬럼 이름
  - nullable(DDL) : null 값 허용여부
  - unique(DDL) : 한 컬럼에 유니크 제약조건을 걸 때 사용. 두 컬럼 이상 유니크 제약조건을 걸려면 클래스에서 @Table.uniqueConstraints 를 사용한다.
  - length(DDL) : String 타입에만 사용. 문자열 길이 제약조건
  - precision, scale (DDL) : BigDecimal(또는 BigInteger) 타입에서 사용. precision은 소수점을  포함한 전체 자릿수, scale은 소수의 자릿수. float, double 타입에는 적용되지 않음.

  @Enumerated
  자바의 enum 타입을 매핑할 때 사용
  - value : 
    EnumTypes.ORDINAL: enum 순서를 DB에 저장.
    EnumTypes.STRING: enum 문자열을 DB에 저장.
   🎈기본값인 ORDINAL인데, 이 상태에서 enum을 수정하게 되면 문제가 생길 수 있으므로, EnumTypes.STRING으로 하는 것이 좋다.

  @Temporal
  날짜타입을 매핑할 때 사용하며 다음의 3가지 종류가 있다.
    @Temporal(TemporalType.DATE)
    private Date date; // 날짜 

    @Temoral(TemporalType.TIME)
    private Date time; // 시간

    @Temoral(TemporalType.TIMESTAMP)
    private Date timestamp; // 날짜와 시간

  @Lob
  DB의 BLOB, CLOB 타입과 매핑된다.
  속성은 따로 없고, 매핑하는 타입이 문자면 CLOB 나머지는 BLOB으로 매핑된다.

  @Transient
  어떤 필드를 매핑하고 싶지 않을 때 사용한다. 

  @Access
  JPA가 엔티티에 접근하는 방식을 지정한다. 
  @Id가 필드에 있으면  AccessType.FILED 로, Getter 메서드에 있으면 AccessType.PROPERTY 로 설정된 것과 같다.
  필드 접근 방식과 프로퍼티 접근 방식을 함께 사용할 수도 있다.

  - AccessType.FILED 
    필드에 직접 접근한다. 접근 권한이 private 해도 접근할 수 있다.

  - AccessType.PROPERTY
    Getter 메서드를 사용하여 메서드를 통해 접근한다.
